# preface —— 编者"整体判断"（卷首，主循环/强模型亲自写）

这一节**不要外包给子agent**——它是整份综述最高价值的部分，由主循环（强模型）通读 sections + critic + crosscut 后亲自写。素材：各主线综述 + critic 暴露的跨主线主轴 + crosscut 的议题矩阵/共性问题/研究空白。

**定位说明：** 编者整体判断是给人深读的**向导和提示**——帮读者快速建立全局视角、知道该优先读什么、注意什么。深度的批判判断仍由人读原文完成。卷首的价值在于"帮人省时间找到最值得深读的点"，而非替代深读。

写成 `data/preface.html`（HTML 片段，会被塞进 `.preface` 容器）。建议结构：

```html
<p class="lead">一段总起：这是什么会、读了多少篇、最强的一个总体感受。</p>
<h3>一、核心成果：……</h3>
<ol class="judge"><li><b>小标题。</b>展开，带具体证据/数字。</li> …</ol>
<h3>二、能力边界：……</h3>
<ol class="judge"> … </ol>
<h3>三、领域站位与争论焦点</h3>
<ol class="judge"><li><b>小标题。</b>点明该领域正在进行的几场"对话/争论"——比如路线分歧、指标共识与分歧、关键假设的对立。各组论文分别站在哪一边，为什么。</li> …</ol>
<h3>四、未来趋势：……</h3>
<ol class="judge"> … </ol>
<p class="kicker">收尾：落到 {AUDIENCE} 身上，最该拿走的几条 + 最该警惕的一条。</p>
```

要点：
- **每条判断都挂具体证据**（哪篇、什么数字），别空泛。
- 优先写 critic 找到的**跨主线主轴**（如"视觉保真 ≠ 物理可用""控制信号粒度谱""效率离实时多远""数据成为新瓶颈"）——这些是单篇看不出、最有洞察的部分。
- 参考 crosscut 的 **issues**（议题矩阵）和 **white_space**（研究空白）来充实"未来趋势"和"能力边界"。参考 **common_problems** 补充"能力边界"中的系统性弱点。
- **Positioning（站位）**：用 crosscut 的 issues 和各主线综述中的 positioning 内容，在卷首层面做一次更高视角的站位总结——这个领域当前有几条主要路线？共识在哪里、分歧在哪里？各路线的关键假设是什么？
- **禁用"不是 X 而是 Y"句式**；平实但专业。
- 诚实标注能力边界与未完成项，别只报喜。
