# crosscut —— 议题对照（子agent，便宜模型，跑一次）

## 定位

这是精读+主题综述/查漏（阶段 5）之后、卷首+组装（阶段 7）之前的阶段。它从全部论文 digest 做一次**系统性对照扫描**，产出跨论文的议题矩阵、共性问题候选、建议二次深读的中心论文、研究空白。

**铁律——"给人的核查清单，AI 不下定论"：**

这一层的全部输出都是"候选线索 + 待人核查"。AI 只做横向扫描、列候选、标注理由，**绝不做最终判断**。真正的深度 critical thinking 由人读原文完成。每一条产出都必须标注"待人核查"。

**为什么用便宜模型：** 这一阶段做的是结构化的横向比对和信息聚合——从 digest 已有字段中提取、分组、交叉引用。不需要高阶推理或原创性洞察（那是人的工作），便宜模型足以胜任。

## 模板

---
你在为一份 {WORKSHOP_NAME} 综述做议题对照扫描。目标读者：{AUDIENCE}。

全部论文精读 digest（JSON 数组，每篇含 id/title/category/problem/method/key_results/limitations/appraisal/future_calls）：
{digests_json}

请做**系统性对照**，严格按下述 JSON schema 输出 `crosscut.json`。

**关键原则（必须遵守）：**
- 你的角色是"横向扫描器"，产出是给人的**核查清单和候选线索**。
- 每一条都是"建议人去核实"的提示，**不是定论**。
- 真正的深度批判判断由人读原文完成，你不越权。
- 无法确认的写"待确认"，不要强行归因或下断语。

请按以下结构输出 JSON：

```json
{
  "issues": [...],
  "common_problems": [...],
  "central_papers": [...],
  "white_space": [...],
  "human_note": "..."
}
```

各字段要求见下文。

## 输出字段说明

### `issues`（跨论文议题 × 论文矩阵）

从全部 digest 中提炼出 5-10 个跨论文的核心议题（如"评估基准的可比性""物理一致性 vs. 视觉保真的权衡""训练数据规模与泛化"等），每个议题列出涉及的论文 id。

**为什么做这个：** 单篇精读看不出横向脉络。把"哪些论文在讨论同一个议题"显式列出来，方便人快速定位并交叉阅读。

```json
{
  "issue": "议题名称（中文，简洁）",
  "description": "这个议题的内涵，一两句话",
  "papers": ["p01", "p05", "p12"],
  "note": "待人核查：为什么这几篇被归在一起、值得关注的张力或分歧"
}
```

### `common_problems`（共性问题候选）

从各篇的 appraisal 和 limitations 中归纳出多篇共有的问题模式（如"多篇缺少人类评估""基准数据集高度重叠""核心指标定义不统一"）。每条列出涉及论文和"为何值得注意"。

**为什么做这个：** 一篇论文的单独局限可能是该领域的系统性弱点。聚合之后这些信号更明显，但它们仍是候选——是否真构成问题，需要人读原文判断。

```json
{
  "problem": "共性问题描述（中文）",
  "papers": ["p03", "p07", "p15"],
  "why_notable": "为什么值得注意（待人核查）"
}
```

### `central_papers`（建议优先二次深读的论文）

从议题覆盖度和方法影响力出发，挑出 3-5 篇建议编者优先二次深读的论文。

**为什么做这个：** 编者时间有限，这里给出一个优先级建议，帮编者分配注意力。但"中心"与否的判断是候选性质，编者可自行调整。

```json
{
  "id": "p05",
  "title": "论文标题",
  "issues_involved": ["议题A", "议题B"],
  "reason": "建议优先读的理由（待人核查）"
}
```

### `white_space`（研究空白）

综合各篇 future_calls 和议题覆盖情况，列出 3-6 条本次 workshop 未充分覆盖但可能重要的研究方向。

**为什么做这个：** 各篇 future_calls 是零散的，汇聚后可能显现出共同指向的空白地带。这些空白是候选——是否真的重要，需要人结合领域知识判断。

```json
{
  "gap": "空白方向描述（中文）",
  "evidence": "依据：哪些篇的 future_calls 指向此处 / 议题矩阵中的缺失",
  "note": "待人核查"
}
```

### `human_note`（给编者的一句话提示）

一句话提示编者这份对照清单的使用方式，例如："以上均为候选线索，建议结合原文逐条核实后取舍。"

## 产出

写入 `data/crosscut.json`。后续由 `build_report.py` 渲染为报告中的"议题 × 论文对照"版块。编者写卷首时也可参考 `crosscut.json` 中的 issues 和 white_space 作为素材。
