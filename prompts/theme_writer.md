# theme_writer —— 一条主线的综述（子agent，每类一个）

模板：

---
你在为一份面向 **{AUDIENCE}** 的领域综述撰写其中一条主线。

- 主线名称：{category}
- 本主线相关论文的结构化提要（JSON 数组，可能为空或很少）：
{digests_json}

请用中文写这一节综述（600–1000 字），要求：

1. **评价优先**：不要平铺直叙地罗列"A 做了 X，B 做了 Y"，而是以评价者的口吻，按各工作**自身范式**判断它们做得好与不好的地方——优缺点都讲。判断要有依据（挂具体数字或实验事实），避免空泛的赞/贬。
2. **核心进展**：本届这条主线的核心推进是什么，用一两句话先给结论。
3. **Positioning（站位与对话）**：点明这条主线内正在进行的"对话"或"争论"——比如方法 A 与方法 B 的路线分歧、评价指标的共识与分歧、同一问题上的不同假设。各篇在这场对话中分别站在什么位置？这些定位比罗列方法更能帮读者理解领域走向。
4. **代表作串联**：点名代表作，串起它们共同的方法取向、以及彼此的分歧/互补。
5. **可落地启发**：给 {AUDIENCE} 的可落地启发。
6. 若本主线论文很少或为空，简要说明，并指出该方向当前的**留白**。

平实但专业，**不要用"不是 X 而是 Y"的句式**。返回纯文本，可用简单小标题（## / ###）。
---

产出顺序必须与 `cats.json` 对齐，写入 `data/sections.json`（list[str]）。
